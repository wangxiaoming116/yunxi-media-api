# 短视频去水印支持哪些平台？抖音快手小红书微博皮皮虾视频号豆包即梦

> **导语**  
> 「到底支持哪些平台」是接入去水印 API 时被问得最多的问题。本文按品类梳理常见覆盖方向，并说明如何用统一 HTTP 接口接入，避免每个平台单独写一套解析。

---

## 为什么平台清单会“变化”

抖音、快手、小红书等平台的页面结构、分享链格式会不定期调整。自建爬虫往往要持续跟版；使用开放 API 时，适配维护在服务端完成，调用方主要关注：

- 统一网关  
- UID / KEY 鉴权  
- 返回 JSON 字段  

云析 API 官网：https://syapi.chuangye.site/  
文档中心：https://syapi.chuangye.site/home/doc  

> 以下为常见方向说明，**最终以文档与实际返回为准**。

---

## 一、短视频去水印（高频）

| 平台 | 典型用途 |
|------|----------|
| 抖音 | 短视频无水印地址、封面/标题等 |
| 快手 | 短视频解析 |
| 小红书 | 视频 / 图文链路 |
| 微博 | 短视频分享链 |
| 皮皮虾 | 短视频解析 |
| 视频号 | 分享链解析（注意合规与可用范围） |
| B 站等 | 部分短视频/相关场景（以文档为准） |

产品编码上，短视频 / 图集 / 实况一类常用 `type=dsp`。

---

## 二、图集与实况

- **图集**：小红书图文、抖音图集等，返回里常见 `pics` 列表  
- **实况 / Live Photo**：适合相册实况、动态图相关场景（以文档说明为准）

做「保存图片 / 看视频」类工具时，建议业务层同时兼容：

1. 有 `playAddr` → 按视频处理  
2. 有 `pics` → 按图集处理  

---

## 三、AI 创作平台分享链

近年来豆包、即梦等内容分享链也常被工具站收录。是否在你账号下可用，请以云析文档中心当前产品说明为准；接入方式仍是同一套 `uid/key/url` 参数。

---

## 四、一个接口覆盖多平台的好处

传统做法：

```text
抖音解析器 + 快手解析器 + 小红书解析器 + ... → 运维成本线性上升
```

开放 API 做法：

```text
业务后端 → 云析 /home/api?type=dsp&uid=&key=&url=分享链接
```

调用方只维护：

- 密钥与点数  
- 超时重试  
- 结果字段映射  

开源调用示例（Python / Node / PHP / Java / curl）：

- https://gitee.com/wzzdq8/yunxi-media-api  
- https://github.com/wangxiaoming116/yunxi-media-api

## 五、最小调用回顾

```bash
curl -G "https://syapi.chuangye.site/home/api" \
  --data-urlencode "type=dsp" \
  --data-urlencode "uid=YOUR_UID" \
  --data-urlencode "key=YOUR_KEY" \
  --data-urlencode "url=任意支持平台的分享链接"
```

免费注册：https://syapi.chuangye.site/user/login/register  

计费常见规则：**解析成功扣点，失败不扣**（以控制台为准），适合先小流量验证再扩容。

## 六、合规提醒

1. 仅解析用户有权处理的内容，遵守平台与当地法律法规  
2. 不要把 API KEY 放进公开前端  
3. 工具产品需在隐私政策中说明数据处理方式
